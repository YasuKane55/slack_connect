# 🎨 ビジュアル＆手応え（ジュース）設計

> 「システムが完璧でも、見た目と さわった手応えが つまらないと 楽しくない」。
> 特にこども向けは **“気持ちよさ”が命**。ここは こだわる前提で設計する。
> 連動：`GAME_PLAN.md`（本格3D/Three.js・ビルボード）／`MINIGAMES.md`

最終更新: 2026-06-11 / v1

## 0. いちばん大事な方針
- **見た目＝スタイライズド（かわいい立体）／手応え＝ジュース全振り**
- フォトリアルは狙わない（重い＆こども端末で動かない）。**低ポリ＋トゥーンで“映える”**
- **ジュース（反応の気持ちよさ）は 3Dの重さと無関係に効く** → 最優先でこだわる
- まず **ルック＆フィール検証プロト**（§7）で 見た目と手応えを 早期に確かめる

---

## 1. アートディレクション（目標の見た目）
- **あつ森っぽい**：やわらかい・あたたかい・まるい。怖くない、ほっとする世界
- **低ポリ＋トゥーン**：少ない面でかわいく。輪郭線で絵本っぽさ
- **配色**：暖色・パステル寄り＋差し色。時間帯で 空と光の色が変化（朝=やわらか／夕=オレンジ／夜=青紫＋街灯）
- **マテリアル**：てかりすぎない（ザラっと/マットに少しだけハイライト）
- **ぷーぶ君**：公式PNGを **ビルボード**で。世界の色味と馴染むよう ふちどり/影を調整

## 2. Three.jsでできること（ブラウザ3Dの“映え”技）
- **トゥーン/セルシェード**（段階的な陰影）＋**輪郭線**（アウトライン）
- **やわらかい影**（PCF/VSMソフトシャドウ）＝立体感の要
- **ふんわり光（ブルーム）**／**色補正（トーンマッピング）**で“映え”
- **環境光遮蔽(AO)**で接地感／**フォグ**で奥行き・空気感
- **水のシェーダー**（池・防潮堤の海）／**草・花のゆれ**（風）
- **インスタンス描画**で 木・花・草を大量に軽く
- **空のグラデ＋時間帯ライティング**（朝昼夕夜で色温度が変わる）
- ※ これらを盛りすぎない。**“ちょうどいい”が一番かわいく見える**

## 3. ぷーぶ君を“生き生き”させる（ビルボードでも魅力的に）
- 🟢 **スクワッシュ＆ストレッチ**：歩く/止まる/ジャンプで ぷにっと伸び縮み
- 🟢 **ぴょこぴょこ歩き**：上下に弾む＋わずかに傾く
- 🟢 **向き**：進行方向で 左右反転／表情差分（PNG複数 or 簡単な切替）
- 🟢 **接地影**：足元に ふんわり丸影（立体世界に馴染む）
- 🟢 **感情表現**：うれしい時 ジャンプ＆キラッ、こまった時 しょんぼり揺れ
- 🟢 **アバター・カスタマイズ**（初期設定§GAME_PLAN1-4）：色/ほっぺ/ぼうし/服/アクセ/表情を
  **パーツ重ね or 色ちがい**で。着せ替えは いつでも変更（見た目で“自分らしさ”）
- ※ ビルボードは“紙人形”だが、上記の動きで **十分かわいく生きて見える**

## 4. インタラクティブな手応え（ジュース）★最重要
**原則**：アクション＝必ず「**音＋見た目の反応＋UIの反応**」をセットで返す。
即時・大げさめ・気持ちよく。失敗も かわいく（罰でなく笑い）。

| アクション | 画面の反応（例） | 音 |
|---|---|---|
| あるく | ぷにぷに弾む・足元にほこりパフ・影が伸縮 | コツコツ/とてとて |
| はなす | 相手が ひょこっと反応・吹き出しポップ・カメラ寄り | ぽよん |
| かう/うる | コインが舞う・プーブ数字がポップ＆カウント・キラッ | チャリン |
| たべる | もぐもぐ＆ほっぺ・湯気/ハート・おなかゲージがピョン | もぐもぐ |
| 水やり | キラキラ水しぶき・植物がふるっと伸びる | しゃー |
| 収穫/採集 | ポップして飛び出す・取得アイテムが手元へ吸い込み | ぽこっ |
| つり | ウキがピクッ→大きな引き・水しぶき・成功でファンファーレ小 | ちゃぽん |
| ペイント/掃除 | こすった跡がパッと色づく・前後で見違える | きゅっきゅ |
| 猫をなでる | 猫が目を細める・ハート・きぶんアップ表示 | にゃー/ごろごろ |
| マンホール/アート発見 | キラーン＋図鑑に“NEW!”スタンプ・小さな祝福 | ぴこーん |
| 称号ゲット | 画面に リボン演出・ぷーぶ君ジャンプ・きらきら | じゃーん |
| 街が発展 | ビフォー→アフターが パッと変わる・住人が拍手 | わぁ |
| ミニゲーム成功 | スコアが弾む・パーティクル・「やったね！」 | ピンポン |

- **共通テク**：イージング(tween)・一瞬の拡大(ポップ)・色フラッシュ・パーティクル・
  数字のカウントアップ・やさしい画面ゆれ（ほんの少し）・ヒットストップ（一瞬止め）
- **触れるものは わかる**：近づくと ふわっと光る/少し跳ねる（インタラクト可能の合図）
- **ホバー/選択**：ボタンや対象が ぷるっと反応（押せる感）

## 5. カメラ・操作感
- **追従カメラ**：少し遅れてついてくる（やわらかい）。ズーム/ピンチ可
- **操作の即応**：入力に対し 遅延なく動き出す（気持ちよさの土台）
- 場面で**軽い演出**：会話で寄る／発見でズーム／イベントで引き
- 酔い対策：急な回転を避け、ゆったり

## 6. 音（SFX/BGM）= 楽しさの半分
- **効果音は必須**：アクションごとに かわいい SE（手応えの核）
- **BGM**：時間帯/エリアで やわらかく変化。うるさくない
- こども向けに **やさしい音色**。音量調整・OFFも可
- ※ 無音だと どんなに動いても 物足りない → 早めに 仮SEを入れる

## 7. 品質と動作のバランス（届く端末で気持ちよく）
- **画質プリセット**：高(PC)/中/低(スマホ)。影・ブルーム・AO・草の量を 段階OFF
- 端末性能を見て **自動選択**＋手動切替
- 低スペックでも **ジュース（音/ポップ/パーティクル少）は維持**＝楽しさは落とさない
- 目標：スマホでも **なめらか（30〜60fps）**

## 8. おすすめの進め方：まず“ルック＆フィール検証プロト”
システム実装より先に、**小さな見た目＆手応えデモ**を作って こだわりを確かめる：
- 内容：低ポリの小さな広場／**ぷーぶ君が ぷにぷに歩く**／やわ影・ブルーム・空の色／
  クリックで **反応する物**を数個（音＋パーティクル＋ポップ）／猫1匹なでられる
- ねらい：「歩くだけで楽しい・さわると気持ちいい」を 早期に体感して 方向性を固める
- これがOKなら、その“気持ちよさ”を 全機能に展開していく
