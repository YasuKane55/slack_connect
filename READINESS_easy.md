## 🙆‍♀️ あなたがやると良いこと（むずかしくない）

### 1. お店や人に「ゲームに出してOK？」と聞く ★いちばん大事
- このゲームは、**本物のお店や人**をモデルにしています。
- だから「こういうゲームを作っていて、あなたのお店を登場させたい」と声をかけて、
  **「いいよ」をもらう**のが先。
- 人の名前・年齢・お話を使うので、本人に確認すると安心。
- → これは早めに動き出すのがおすすめ（作るより前でもOK）。

### 2. ぷーぶ君の絵を「パーツごと」に分けて用意
- 体／目（いろんな表情）／手／ほっぺ／帽子…を、**バラバラの画像**にして渡す。
- バラバラだと「色を変える・帽子をかぶせる」みたいな**着せ替え**ができる。
- 1枚の絵だと着せ替えできないので、パーツ分けが大事。

### 3. キャラのしゃべり方（大阪弁の濃さ）を決める
- 各キャラを **1（うすい）〜3（こてこて）** で選ぶだけ。
- 例：「ミンさん＝3、ミッキーさん＝1」みたいに番号を言ってくれればOK。

### 4. 「まずどの遊びを濃く作る？」を1つ選ぶ
- お店屋さん／農業／ものづくり…ぜんぶ一気は大変。
- **まず1本**に絞ると、早く“遊べる形”になる。

> この4つだけ、あなたが進めてくれたら大丈夫。あとは作っていけます。

## 🛠 どんな順番で作る？（イメージ）
1. 土台づくり（家でいう基礎工事）
2. **まちを歩けるようにする**（デモの気持ちよさを本番へ）
3. **自分のぷーぶ君をつくる画面**（色・ほっぺ・帽子・なまえ・誕生日）
4. **会話・お買い物・もちもの**
5. **時間（朝昼夕夜）と、おなか・ねむけ・きぶん**
6. **クエスト（困りごと）と称号**
7. **採集・つり・農業・ものづくり（ミニゲーム）**
8. **部屋づくり・自分のお店・まちの発展**
9. **図鑑・ひみつの農園・仕上げ（音や演出）**
10. **オンライン（友達と遊ぶ）**

## 📖 ことば辞典（勉強用・専門用語をやさしく）

| ことば | やさしい意味 |
|---|---|
| **プロトタイプ / デモ** | お試しで作った“動く見本”。今あるやつ。 |
| **Three.js（スリージェイエス）** | ブラウザで3D（立体）を表示する道具（ライブラリ）。 |
| **ライブラリ** | 便利な機能の“道具セット”。自分で全部作らず借りる。 |
| **ビルド** | バラバラのプログラムを、配れる形に“組み立てる”こと。 |
| **CDN** | ネット上の倉庫から道具を読み込むしくみ。準備がかんたん。 |
| **アセット** | 絵・音・3Dモデルなどの“素材”のこと。 |
| **ビルボード** | 平面の絵を、いつもこっちを向くように立てる方法（紙人形みたいに）。ぷーぶ君に使用。 |
| **スプライト** | 上の「平面の絵」そのもの。 |
| **当たり判定（コリジョン）** | 壁や建物に“ぶつかって通れない”しくみ。 |
| **座標 / グリッド** | 地図の上の“位置（縦・横の数字）”。マス目で場所を決める。 |
| **NPC** | プレイヤー以外の、まちの人（お店の人や通行人）。 |
| **セーブ / localStorage** | 遊んだ記録の保存。localStorageは“その端末の中に保存”する箱。 |
| **ジュース（演出の気持ちよさ）** | さわった時の音・キラキラ・はずむ動きなど、“楽しい手応え”の総称。 |
| **図鑑（ずかん）** | 集めたもの（魚・虫・アート・マンホール等）を記録するコレクション帳。 |
| **プリセット** | あらかじめ用意した“選ぶだけ”の組み合わせ。 |
| **画質プリセット** | 「キレイ重視／軽さ重視」など、見た目の重さを切り替える設定。 |
| **fps（エフピーエス）** | 1秒間のパラパラ漫画の枚数。多いほど“なめらか”。 |
| **サーバー** | みんなのデータをまとめる“ネット上のコンピュータ”。オンラインで使う。 |
| **リアルタイム同期** | 友達のキャラの動きを“今この瞬間”で合わせて見せること。 |
| **訪問モデル** | 「友達のまちに遊びに行く」方式。まちは一人ひとり別、というこのゲームの考え方。 |
| **GitHub（ギットハブ）** | プログラムやファイルを保管・共有する場所（ネット上）。 |
| **リポジトリ** | GitHub上の“プロジェクトの保管箱”。今回の `slack_connect`。 |
| **ブランチ** | 保管箱の中の“作業用の枝分かれ”。`main` は本線。 |
| **コミット / プッシュ** | 変更を記録（コミット）して、ネットに反映（プッシュ）すること。 |
| **GitHub Pages** | GitHubの中身を“Webページとして公開”できる機能。 |
| **URL** | ページの住所（https://…）。これを送れば友達も開ける。 |

> わからない言葉が出てきたら、いつでも「これどういう意味？」と聞いてください。
> その都度かみくだいて説明します🙂
