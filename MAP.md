## 0. 地図づくりの方針
- **配置ルール（ユーザー指定・最優先）は §1-2 にまとめ**。以下の思想より優先する
- 中心に **みんなのうえん（広い敷地）＋その中の こどもプーブリカ**。そこから街が広がる
- **エリアはゆるいテーマ分けに留める**：基本は **住宅街の中に お店が混然と配置**される。
  「だいたいこの辺は食が多い」程度の ゆるい傾向で、**厳格な区画分けはしない**
- 歩いて回れるサイズ。**材料を売る店は、作る人の近く**に置く（動線が自然）
- 北加賀屋の“商店街＋ものづくりの街＋路地と長屋”の空気感をオマージュ
- 低ポリ3D／斜め見下ろし（GAME_PLAN参照）。建物は箱＋屋根の仮モデルから
- **街じゅうに アート作品** が点在（プーブタウンの特徴・§2-2）
- **街に猫が多い**。人気者の白灰猫 **ウルフ** ＋ 各拠点の **主猫**（§2-2）

> ✅ **決定：遊びやすい「ゆるテーマ＋住宅街に混然」配置でいく**（実際の地理再現はしない）。
> 厳密なゾーンではなく、住宅街にお店が点在し、ゆるい傾向で“らしさ”を出す。

- ⚠️ 上図の「西/東/南」は **厳密な区画ではなく ゆるい傾向**。実際は住宅街に混然と配置
- **中央＝みんなのうえんの広い敷地**：その中に **こどもプーブリカ(HUB)** と 広場。
  畑・農具/肥料店も同じ敷地まわり。進行・案内・称号・朝市/イベント会場

- **ものづくり寄り**：アーティスト＆材料店（作る人の近くに材料）
- **くらし寄り**：買い物・美容・本・休憩＋住宅が多いあたり（街で一番にぎやか）

## 1-2. 配置ルール（ユーザー指定・最優先）

> 配置は基本おまかせだが、**次のルールは必ず守る**。

1. **ゆるい振り分け＋住宅街に混然**：エリアはテーマの傾向だけ。厳格に分けない。
   お店は住宅の合間に点在する（路地・長屋の街）
2. **こどもプーブリカ ＝ みんなのうえんの広い敷地の中の建物**。
   ハブと畑が一体。敷地内に 広場・畑区画・(近くに)農具/肥料 も置く
3. **長屋（1棟・3軒つながり）**：`nagaya_A`
   - トークウィズ チャイ屋 ／ ナチュラルサンド ／ 本のすみか
4. **2階建て（1棟・1フロア2軒＝計4軒）**：`bldg_2f_B`
   - 1F：イエコーヒー ＋ サンセール
   - 2F：エトラガラス ＋ 美容室シー
   - ※ 2Fへは階段。内部や入口の扱いは §5 で要検討（プロトは入口で店選択でも可）
5. **スーパースタジオ ＝ 改修した巨大倉庫・2階建て**。**フットプリント超大型**。
   - **1階**：木の彫刻(モク)／屋台作家(ヤタ)／パフォーミングアーツ(マイム)／巨大彫刻(ダイナ)＋夜DJスペース
   - **2階**：映像(エイ)／現代絵画(いろ)／写真(パシャ)／グラフィック(ぐらこ)／WEB(ウェブ)／建築(ケンチ)
   - ※ 大きい作業（彫刻・屋台・パフォーマンス・巨大彫刻）が1階、その他が2階
6. **プーブ商店街 ＝ かなり大きい アーケード商店街**（屋根付き通り）。**ブティックオキ**が
   この中にある。**他のくらし系のお店も この中に入れてよい**（商店街らしいにぎわい）。
   屋根があるので 雨でも歩ける／中は通路＋両側に店という構造

## 2-2. 街の特徴：アート作品と猫 🐈🎨

プーブタウンらしさを決める2大要素。**配置物＋コレクション＋なごみ**として実装。

### 🎨 街なかアート作品
- 街じゅうに **アート作品が点在**（彫刻・壁画・オブジェ）。歩くたびに出会える
- **作者は街の人**：ダイナさん(巨大彫刻)/モクさん(木彫)/いろちゃん(絵)/アンドソー(壁画)など
  → 「これ、◯◯さんの作品だ」と分かると街と人がつながる
- ゲーム機能：
  - 🔎 **アート図鑑/アート巡り**：見つける・写真にとる（パシャちゃんと連携）コレクション
  - 😊 そばに行くと きぶんUP・小ネタ／さんぽや・わたるが在りかのヒント
  - 🏅 全部めぐると称号（例「アートはかせ」）
  - 🎪 イベント「**まちじゅうアート展**」：新作が増える日（街が更新＝長く遊べる）

### 🐈 猫たち
- **街に猫が多い**。なでられる・ついてくる・ひなたぼっこ等でなごむ存在
- ⭐ **ウルフ**：**白灰色の人懐っこい猫**。街じゅうを歩き回る、みんなに愛される
  アイドル的存在。**固定位置なし＝出会えるとうれしい**（花屋台ふと同じ移動ロジック）。
  なでると **きぶん大UP**、仲良くなると ついてきてくれる など
- 🏠 **拠点の主猫（ぬし猫）**：みんなのうえん・スーパースタジオ等、各拠点に1匹いる
  「その場所の主」。看板猫的に居ついている
- ゲーム機能：
  - 🐾 **ねこ図鑑**：出会った猫を記録（種類・場所）
  - 😊 なでる＝きぶん回復／好物(にぼし等)をあげると仲良し度UP
  - 📜 猫クエスト：「まいごの子猫」「ウルフを さがそう」「拠点の主猫に エサを」
  - こども向けに かわいくて安心（こわくない）

## 3. 実装に必要なデータ構造（スキーマ案）

実装時はJSON等で持つ想定。座標は「グリッド（タイル）」で管理し、3Dに変換。

### 3-1. ワールド
```js
world = {
  size: { w: 64, h: 64 },        // タイル数(仮)。要決定
  tileSize: 2,                    // 1タイル=2m(仮)
  groundZones: [                  // 地面の種類をエリアで塗る
    { type: "plaza",  rect:{x,z,w,h} },
    { type: "road",   path:[...] },
    { type: "grass",  rect:{...} },
    { type: "water",  rect:{...} },  // 池/川(通行不可)
    { type: "sand",   rect:{...} },  // はまべ
    { type: "soil",   rect:{...} },  // 畑
  ],
  districts: [ { id:"east_food", name:"たべもの横丁", rect:{...} }, ... ],
  decorations: [ {type:"tree|lamp|bench|sign", pos, rot} ],  // インスタンス描画
  cameraBounds: {...},
  playerSpawn: { pos:{x,z}, dir },   // 初回=プレイヤーの家の前
}
```
- 地面：タイルコードの2D配列でも、ゾーン矩形でもOK（プロト＝ゾーン矩形が楽）

### 3-2. 建物（building）※マルチテナント対応
建物は **複数の店(tenant)** を持てる。単独店は tenant 1つ、長屋/2階建て/商店街は複数。
```js
building = {
  id, name,
  kind: "single|nagaya|multi_floor|arcade|warehouse|site",  // §1-2のまとまり種別
  footprint: { x, z, w, d }, rotation,                       // 倉庫/商店街/敷地は大きめ
  visual: { placeholder, color, signText },
  tenants: [                       // ← 中に入る店(1つ以上)
    {
      shopId, name, ownerIds:[npcId...],
      floor: 1,                     // 2階建て用(1/2)。長屋・商店街は1
      unitPos: {x,z},               // 建物内での区画位置(長屋の並び/商店街の左右)
      door: { pos:{x,z}, facing },  // その店の入口
      openHours: ["asa","hiru",...],
      interior: { type:"panel|scene|outdoor", layoutId },
      enterAction: "openShopPanel | loadInterior | none",
    }, ...
  ],
}
```
- 例）`nagaya_A`：tenants=[チャイ屋, ナチュラルサンド, 本のすみか]（floor:1, unitPosで横並び）
- 例）`bldg_2f_B`：tenants=[イエ(F1),サンセール(F1),エトラ(F2),美容室シー(F2)]
- 例）`arcade_puub`：屋根付き通路の両側に複数tenant（ブティックオキ＋他くらし系）

## 4. 配置・サイズの決め方（実装メモ）
- **建物フットプリント目安**（タイル）：小店 3×3 ／ 中店 4×4 ／ 長屋 3×9(3軒) ／
  2階建て 5×5(各階2軒) ／ **プーブ商店街(アーケード)** 6×20〜(細長い通路型) ／
  **スーパースタジオ(巨大倉庫)** 10×12〜 ／ **みんなのうえん敷地(site)** 14×14〜(畑＋HUB＋広場)
- **道**：各エリアと中央広場を道でつなぐ。道幅2タイル。交差点に街灯/看板
- **入口ルール**：建物の道に面した側に `door` を1つ。プレイヤーが door タイルで
  アクション → `enterAction`（パネル or 内部シーン）
- **コリジョン**：建物フットプリント・木・水は通行不可。プロトは **AABB当たり判定**でOK
- **採集/つりポイント**：park/pond/beach 内に「採集ノード」を点在（時間帯で出現変化）
- **花屋台ふ 出現ノード**：各エリアに2〜3か所、計8〜12点ほど

## 5. 内部（インテリア）の扱い ― 要決定
プロトを軽くするための方針案（建物ごとに `interior.type` で指定）：
- **panel（推奨・多数）**：入口で **お店UI/会話パネル**を開くだけ（内部3Dなし）。
  物販店・材料店はこれで十分軽い
- **scene（重要拠点のみ）**：簡単な内部3Dに切替。
  対象案：**プレイヤーの家**(部屋づくり)／**こどもプーブリカ**(ハブ)／
  **スーパースタジオ**(WS)／**エトラガラス**(WS)／**自分のお店**／
  雰囲気重視店(オハラ夜・純喫茶こはく・チャイ屋・本のすみか・美容室)
- **outdoor**：広場・公園・池・はまべ・畑（屋外のまま操作）
