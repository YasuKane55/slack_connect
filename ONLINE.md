## ① リアルタイム・キャラ同期（みんなで同じ街を歩く）

### 結論
**実装できる。** 戦闘も高速物理もない のんびり系なので、ネット同期の中では やさしい部類。
ただし **オンラインは全機能の中で一番むずかしい**ので、**段階的**に作る。

### しくみ（小規模・訪問ベース）
- 世界全体は共有しない。**1セッション＝「ホストの街」＋訪問者 数人（例 4〜8人）**
  → 同期する人数が少なく、**軽くてシンプル**
- 各端末は「**自分の 位置・向き・歩く/アクション**」をサーバーへ送る
- サーバーは 同じ街にいる人へ **ブロードキャスト**。他人キャラは **補間（lerp）でなめらか表示**
- **時刻はサーバーが配信**（朝昼夕夜は全員同期）
- 送る頻度：位置は 1秒に約10回程度＋補間。会話/購入などは イベントで送信

### 何を同期する / しない
- 同期：プレイヤーの 位置・向き・歩行/しぐさ、チャット、共同クエストの進行、時刻
- 同期しない（各自）：発展度・部屋・畑・お店・図鑑など **個人の街データ**
  （訪問中は ホストの街データを読み込んで表示）

### 技術の選択肢
- **既製リアルタイム基盤を推奨**（ネットコード自作は大変）：
  - **Colyseus**（Node製・部屋管理が楽・JS/TS）／**Nakama**／**Photon**／
    **Supabase Realtime** など
- 自作する場合：Node.js + `ws`（WebSocket）+ クライアント補間。プロト検証向き
- 認証/フレンド：まずは **合言葉/ルームコード**で友達の街に入る簡単方式から

### 正直な注意点（期待値）
- 遅延は必ず出る → **ゆるい補間**で十分自然に見える（この genre はシビアでない）
- チート対策・通信費・同時接続数 はオンライン化で初めて出る課題 → 小規模設計で軽減
- **まず1人用を完成 → 段階的にオンライン**（下の順番）

### 段階的プラン
1. 1人用ゲーム完成（土台）
2. **訪問＋位置同期だけ**（友達の街に入って一緒に歩ける）
3. かんたんチャット（定型スタンプ/絵文字＝子ども安全）
4. お店で売り買い・部屋を見せ合う
5. **ライト共同クエスト**（③）
6. ゲストルーム招待（②）

## ② ゲストルーム招待（友達の部屋を自分の世界に）

> 友達の **部屋（インテリア）＋キャラの見た目** を、自分の街に **コピーして置ける** 機能。
> リアルタイム同期より簡単（**データのコピー**だから）。

### しくみ
- 友達の許可で、その子の **部屋レイアウト（家具リスト＋配置）** と **キャラ見た目** を
  **スナップショット保存**
- 自分の街の **「ゲストハウス」**（空き家を再生した建物など）に **再現**
- そこに **ゲストNPC（友達のキャラのコピー）** が居て、あいさつ・短い会話ができる
- ※ **静的コピー**：友達が後で模様替えしても 自動では変わらない（“その時の記念”）
- 複数の友達を招待して **ゲストハウス長屋**にする展開も

### うれしいポイント
- オフラインでも 友達の部屋を眺められる＝**いつでも会える感**
- 「この家具どこで買ったの？」など **交流・あこがれ・参考**に（ねらい§0）
- コレクション要素：招待した友達の数／もらった記念品

### データ（イメージ）
```js
guestRoom = {
  ownerName, snapshotDate,
  character: { appearance },          // 見た目のコピー
  layout: [ {itemId, pos, rot}... ],  // 家具配置のコピー
  greetings: [...],                   // ゲストNPCの一言
}
```

## ③ ライト共同クエスト（称号なし・気軽に・くり返しOK）

> 友達と訪問中に **その場でサッと** 遊べる軽いやつ。**称号なし**、報酬は 少しのプーブ＋
> きぶんUP＋ちょっとしたアイテム。勝ち負けより **一緒に楽しい** を重視。

| ID | タイトル | 内容 | 人数 | 報酬(ライト) |
|---|---|---|---|---|
| L1 | ふたりで たからさがし | 街の かくれた物（マンホール/アート/かくれ猫）を 協力して 制限時間で探す | 2〜 | 少しプーブ＋きぶん |
| L2 | むしとり／つり たいかい | 時間内に みんなで たくさん 集める（**合算**・勝ち負けなし） | 2〜 | 集めた数で プーブ山分け |
| L3 | ウルフと かくれんぼ | 逃げる ウルフ（猫）を みんなで さがす／追う | 2〜 | にぼし＋きぶん |
| L4 | おつかい リレー | 荷物を 分担して 複数のお店へ 同時配達（協力すると速い） | 2〜 | プーブ＋お店のおまけ |
| L5 | みんなで しゅうかく | 畑/果樹を ふたりで 一気に 世話・収穫 | 2〜 | とれた野菜/果物を 山分け |

### 共通ルール（ライト設計）
- **すぐ始められて すぐ終わる**（数分）。失敗してもOK・やり直し自由（急かさない§0）
- **協力で得（合算・分担）**＝競争じゃなく一緒だと楽しい/お得
- 報酬は **小さめ**（メインの進行＝個人クエストを邪魔しない）
- ホストの街で発生。訪問者も気軽に参加/離脱できる
- 1人でも 簡易版で遊べる（ソロ時は NPC が相手）と尚よい

### これから詰めたいこと
- 各ライトクエストの 具体ルール・制限時間・報酬量
- チャット/合図の安全設計（定型・スタンプ中心）
- ルームコード or フレンド管理の方式
- リアルタイム基盤の選定（Colyseus 等）と プロト検証
